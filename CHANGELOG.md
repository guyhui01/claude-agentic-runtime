### 🐛 Fixed
- **WF-009 — the two decision gateways were fooled by an honest in-band refusal (hardened after the first live run).** The first billed live run (Sonnet 5) returned `completed` 6/6, but a **content** inspection (not just verdicts) exposed a **hollow pass**: the seed carried a recruitment brief but no candidate CVs, so STEP-04's active sourcing had nothing to work on — the agent honestly returned a 3-entry **placeholder** shortlist ("… only to satisfy `minItems:3`") and `selectedCandidate = "None — no candidate can be selected"`. The gateways `rh-shortlist-validated` (`minArrayLen ≥ 3`) and `sel-candidate-selected` (`nonEmptyString`) **passed on that content** — the exact `NO` branch they exist to halt on — cascading to a hollow offer letter (`"CANNOT BE ISSUED"`). Hardened: `rh-shortlist-validated` now counts REAL candidates (`countAffirmativeField(shortlist, "candidate") ≥ 3`) and `sel-candidate-selected` requires an `affirmativeString`, so both fail closed on the refusal (regression tests reproduce the live outputs in `test/spine-wf-009.test.ts`). Hardening the two gateways suffices — once the valves fail closed, the downstream completeness criteria only ever see real content. The hollow live trace is **not versioned** (it would be theater). **Live-verified 2026-07-12** (billed, Sonnet 5): re-running the same no-CV seed now **halts fail-closed at STEP-04** (`rh-shortlist-validated`) on the agent's honest refusal ("No candidate data available") — the identical content that produced the hollow pass before hardening — a "gate says no" proof on the candidate-selection decision; anonymized trace versioned `docs/audit/live-runs/wf-009-live-result-gate-halt.json` (renamed to preserve it alongside the later end-to-end `completed` run — see Added; secret-scan clean, no model field).

### Added

  - **`schema/sidecar.schema.json`** (JSON Schema 2020-12) : index machine-lisible du catalogue, encodant de façon **exécutable** 5 des 7 caractéristiques ISO/IEC 25012 retenues (ADR-0006) — complétude, cohérence (`if/then` par type), crédibilité (`source` = `{file, catalogTag}`), actualité (tag épinglé + horodatage), exactitude/accessibilité bien-formées. `path`/`source.file` anti-traversal (pas de `/` initial ni `..`).

  - **`src/sidecar/integrity.ts`** : les 2 caractéristiques ISO 25012 hors portée d'un JSON Schema — `checkReferentialIntegrity` (`dependsOn`→`id`), `checkAccessibility` (joignabilité disque, refus défensif des chemins absolus/`..`) — plus `checkUniqueIds` (unicité globale des `id`) et l'agrégat `checkIntegrity`.

Comble le vide que le SDK ne couvre pas — la cohérence I/O inter-étapes. 7 tests.
